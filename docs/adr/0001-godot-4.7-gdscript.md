# ADR-0001: Use Godot 4.7 with GDScript

- **Status:** Accepted
- **Date:** 2026-06-24
- **Deciders:** Asger (solo)

## Context

Kiosk is a 2D, bit-style management-incremental (see `CONTEXT.md`). It is built **solo, with
AI assistance, hobby-first**, with a possible path to commercial later. The project is
committed to the Godot engine; the open choices were the engine *version* and the *scripting
language* (GDScript vs. C#). Target is **desktop-first with mobile in mind**, keeping a web
(HTML5) export viable as a feedback build.

Key forces:

- **Iteration speed** matters most for a solo hobby dev maintaining momentum.
- **AI/tutorial coverage** strongly favors GDScript — most Godot examples, forum answers, and
  training data are GDScript, making AI assistance more reliable.
- **Performance** is irrelevant at this game's scale (a 2D management/idle sim).
- **Export friction**, especially for web, is lower with GDScript.

## Decision

Use **Godot 4.7** (latest stable, released 2026-06-18) with **GDScript** as the sole
scripting language for v1.

## Consequences

**Positive**

- Fast iteration: hot reload, no compile step.
- Most reliable AI assistance and the largest body of copy-adaptable examples.
- Smooth path to a web export for sharing playtest builds.
- Mobile export remains available from the same project.

**Negative / accepted trade-offs**

- Weaker static typing than C# (mitigated by GDScript's optional typing — use it).
- If the codebase ever grows very large or needs a CPU-heavy system, a targeted C# module may
  be reconsidered (Godot supports mixing); that would warrant a new ADR.

**Neutral**

- Tied to Godot 4.x conventions and 4.7 features; upgrading minor versions is low-risk.

## Alternatives considered

- **Godot 4.6.3 + GDScript** — a slightly more battle-tested point release. Rejected only
  because starting fresh on the latest stable is fine and avoids an early upgrade.
- **Godot 4.7 + C#** — stronger typing/tooling, but heavier setup, slower iteration, weaker
  Godot-specific AI/example coverage, and more web-export friction. Overkill for this scale.
- **GDScript now, C# later as needed** — viable hedge, but mixing adds complexity unlikely to
  be needed; revisit via a new ADR only if a concrete system demands it.
